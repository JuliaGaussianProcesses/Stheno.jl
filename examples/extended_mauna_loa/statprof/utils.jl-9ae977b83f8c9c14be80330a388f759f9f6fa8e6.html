33 """
34 ColVecs(X::AbstractMatrix)
35
36 A lightweight wrapper for an `AbstractMatrix` which interprets it as a vector-of-vectors, in
37 which each _column_ of `X` represents a single vector.
38
39 That is, by writing `x = ColVecs(X)`, you are saying "`x` is a vector-of-vectors, each of
40 which has length `size(X, 1)`. The total number of vectors is `size(X, 2)`."
41
42 Phrased differently, `ColVecs(X)` says that `X` should be interpreted as a vector
43 of horizontally-concatenated column-vectors, hence the name `ColVecs`.
44
45 ```jldoctest
46 julia> X = randn(2, 5);
47
48 julia> x = ColVecs(X);
49
50 julia> length(x) == 5
51 true
52
53 julia> X[:, 3] == x[3]
54 true
55 ```
56
57 `ColVecs` is related to [`RowVecs`](@ref) via transposition:
58 ```jldoctest
59 julia> X = randn(2, 5);
60
61 julia> ColVecs(X) == RowVecs(X')
62 true
63 ```
64 """

99 RowVecs(X::AbstractMatrix)
100
101 A lightweight wrapper for an `AbstractMatrix` which interprets it as a vector-of-vectors, in
102 which each _row_ of `X` represents a single vector.
103
104 That is, by writing `x = RowVecs(X)`, you are saying "`x` is a vector-of-vectors, each of
105 which has length `size(X, 2)`. The total number of vectors is `size(X, 1)`."
106
107 Phrased differently, `RowVecs(X)` says that `X` should be interpreted as a vector
108 of vertically-concatenated row-vectors, hence the name `RowVecs`.
109
110 Internally, the data continues to be represented as an `AbstractMatrix`, so using this type
111 does not introduce any kind of performance penalty.
112

114 julia> X = randn(5, 2);

119 true
