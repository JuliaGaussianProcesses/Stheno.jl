2975 # Goal: Hash approximately log(N) entries with a higher density of hashed elements
2976 # weighted towards the end and special consideration for repeated values. Colliding
2977 # hashes will often subsequently be compared by equality -- and equality between arrays
2978 # works elementwise forwards and is short-circuiting. This means that a collision
2979 # between arrays that differ by elements at the beginning is cheaper than one where the
2980 # difference is towards the end. Furthermore, choosing `log(N)` arbitrary entries from a
2981 # sparse array will likely only choose the same element repeatedly (zero in this case).
2982
2983 # To achieve this, we work backwards, starting by hashing the last element of the
2984 # array. After hashing each element, we skip `fibskip` elements, where `fibskip`
2985 # is pulled from the Fibonacci sequence -- Fibonacci was chosen as a simple
2986 # ~O(log(N)) algorithm that ensures we don't hit a common divisor of a dimension
2987 # and only end up hashing one slice of the array (as might happen with powers of
2988 # two). Finally, we find the next distinct value from the one we just hashed.

2991 # with linear indices, but `findprev` uses `keys`. Hoist out the conversion "maps":
