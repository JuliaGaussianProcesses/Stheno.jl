55 """
56 ==(x, y)
57
58 Generic equality operator. Falls back to [`===`](@ref).
59 Should be implemented for all types with a notion of equality, based on the abstract value
60 that an instance represents. For example, all numeric types are compared by numeric value,
61 ignoring type. Strings are compared as sequences of characters, ignoring encoding.
62 For collections, `==` is generally called recursively on all contents,
63 though other properties (like the shape for arrays) may also be taken into account.
64
65 This operator follows IEEE semantics for floating-point numbers: `0.0 == -0.0` and
66 `NaN != NaN`.
67
68 The result is of type `Bool`, except when one of the operands is [`missing`](@ref),
69 in which case `missing` is returned
70 ([three-valued logic](https://en.wikipedia.org/wiki/Three-valued_logic)).
71 For collections, `missing` is returned if at least one of the operands contains
72 a `missing` value and all non-missing values are equal.
73 Use [`isequal`](@ref) or [`===`](@ref) to always get a `Bool` result.
74
75 # Implementation
76 New numeric types should implement this function for two arguments of the new type, and
77 handle comparison to other types via promotion rules where possible.
78
79 [`isequal`](@ref) falls back to `==`, so new methods of `==` will be used by the
80 [`Dict`](@ref) type to compare keys. If your type will be used as a dictionary key, it
81 should therefore also implement [`hash`](@ref).
82
83 If some type defines `==`, [`isequal`](@ref), and [`isless`](@ref) then it should
84 also implement [`<`](@ref) to ensure consistency of comparisons.

88 """
89 isequal(x, y)
90
91 Similar to [`==`](@ref), except for the treatment of floating point numbers
92 and of missing values. `isequal` treats all floating-point `NaN` values as equal
93 to each other, treats `-0.0` as unequal to `0.0`, and [`missing`](@ref) as equal
94 to `missing`. Always returns a `Bool` value.
95
96 `isequal` is an equivalence relation - it is reflexive (`===` implies `isequal`), symmetric
97 (`isequal(a, b)` implies `isequal(b, a)`) and transitive (`isequal(a, b)` and
98 `isequal(b, c)` implies `isequal(a, c)`).
99
100 # Implementation
101 The default implementation of `isequal` calls `==`, so a type that does not involve
102 floating-point values generally only needs to define `==`.
103
104 `isequal` is the comparison function used by hash tables (`Dict`). `isequal(x,y)` must imply
105 that `hash(x) == hash(y)`.
106
107 This typically means that types for which a custom `==` or `isequal` method exists must
108 implement a corresponding [`hash`](@ref) method (and vice versa). Collections typically
109 implement `isequal` by calling `isequal` recursively on all contents.
110
111 Furthermore, `isequal` is linked with [`isless`](@ref), and they work together to
112 define a fixed total ordering, where exactly one of `isequal(x, y)`, `isless(x, y)`, or
113 `isless(y, x)` must be `true` (and the other two `false`).
114
115 Scalar types generally do not need to implement `isequal` separate from `==`, unless they
116 represent floating-point numbers amenable to a more efficient implementation than that
117 provided as a generic fallback (based on `isnan`, `signbit`, and `==`).
118
119 # Examples
120 ```jldoctest
121 julia> isequal([1., NaN], [1., NaN])
122 true
123
124 julia> [1., NaN] == [1., NaN]
125 false
126
127 julia> 0.0 == -0.0
128 true
129
130 julia> isequal(0.0, -0.0)
131 false
132
133 julia> missing == missing
134 missing
135
136 julia> isequal(missing, missing)
137 true
138 ```

149 """
150 isless(x, y)
151
152 Test whether `x` is less than `y`, according to a fixed total order (defined together with
153 [`isequal`](@ref)). `isless` is not defined on all pairs of values `(x, y)`. However, if it
154 is defined, it is expected to satisfy the following:
155 - If `isless(x, y)` is defined, then so is `isless(y, x)` and `isequal(x, y)`,
156 and exactly one of those three yields `true`.
157 - The relation defined by `isless` is transitive, i.e.,
158 `isless(x, y) && isless(y, z)` implies `isless(x, z)`.
159
160 Values that are normally unordered, such as `NaN`,
161 are ordered after regular values.
162 [`missing`](@ref) values are ordered last.
163
164 This is the default comparison used by [`sort`](@ref).
165
166 # Implementation
167 Non-numeric types with a total order should implement this function.
168 Numeric types only need to implement it if they have special values such as `NaN`.
169 Types with a partial order should implement [`<`](@ref).
170 See the documentation on [Alternate orderings](@ref) for how to define alternate
171 ordering methods that can be used in sorting and related functions.
172
173 # Examples
174 ```jldoctest
175 julia> isless(1, 3)
176 true
177
178 julia> isless("Red", "Blue")
179 false
180 ```

188 """
189 isgreater(x, y)
190
191 Not the inverse of `isless`! Test whether `x` is greater than `y`, according to
192 a fixed total order compatible with `min`.
193
194 Defined with `isless`, this function is usually `isless(y, x)`, but `NaN` and
195 [`missing`](@ref) are ordered as smaller than any ordinary value with `missing`
196 smaller than `NaN`.
197
198 So `isless` defines an ascending total order with `NaN` and `missing` as the
199 largest values and `isgreater` defines a descending total order with `NaN` and
200 `missing` as the smallest values.
201
202 !!! note
203
204 Like `min`, `isgreater` orders containers (tuples, vectors, etc)
205 lexicographically with `isless(y, x)` rather than recursively with itself:
206
207 ```jldoctest
208 julia> Base.isgreater(1, NaN) # 1 is greater than NaN
209 true
210
211 julia> Base.isgreater((1,), (NaN,)) # But (1,) is not greater than (NaN,)
212 false
213
214 julia> sort([1, 2, 3, NaN]; lt=Base.isgreater)
215 4-element Vector{Float64}:
216 3.0
217 2.0
218 1.0
219 NaN
220
221 julia> sort(tuple.([1, 2, 3, NaN]); lt=Base.isgreater)
222 4-element Vector{Tuple{Float64}}:
223 (NaN,)
224 (3.0,)
225 (2.0,)
226 (1.0,)
227 ```

229 # Implementation
230 This is unexported. Types should not usually implement this function. Instead, implement `isless`.

556 function _extrema_itr(f, itr)
557 y = iterate(itr)
558 y === nothing && throw(ArgumentError("collection must be non-empty"))
559 (v, s) = y
560 vmin = vmax = f(v)
561 while true
562 y = iterate(itr, s)
563 y === nothing && break
564 (x, s) = y
565 fx = f(x)
566 vmax = max(fx, vmax)
567 vmin = min(fx, vmin)
568 end
569 return (vmin, vmax)
570 end

1320 in(item, collection) -> Bool

1322
1323 Determine whether an item is in the given collection, in the sense that it is
1324 [`==`](@ref) to one of the values generated by iterating over the collection.
1325 Returns a `Bool` value, except if `item` is [`missing`](@ref) or `collection`
1326 contains `missing` but not `item`, in which case `missing` is returned
1327 ([three-valued logic](https://en.wikipedia.org/wiki/Three-valued_logic),
1328 matching the behavior of [`any`](@ref) and [`==`](@ref)).
1329
1330 Some collections follow a slightly different definition. For example,
1331 [`Set`](@ref)s check whether the item [`isequal`](@ref) to one of the elements.
1332 [`Dict`](@ref)s look for `key=>value` pairs, and the key is compared using
1333 [`isequal`](@ref). To test for the presence of a key in a dictionary,
1334 use [`haskey`](@ref) or `k in keys(dict)`. For these collections, the result
1335 is always a `Bool` and never `missing`.
1336
1337 To determine whether an item is not in a given collection, see [`:∉`](@ref).
1338 You may also negate the `in` by doing `!(a in b)` which is logically similar to "not in".
1339
1340 When broadcasting with `in.(items, collection)` or `items .∈ collection`, both
1341 `item` and `collection` are broadcasted over, which is often not what is intended.
1342 For example, if both arguments are vectors (and the dimensions match), the result is
1343 a vector indicating whether each value in collection `items` is `in` the value at the
1344 corresponding position in `collection`. To get a vector indicating whether each value
1345 in `items` is in `collection`, wrap `collection` in a tuple or a `Ref` like this:
1346 `in.(items, Ref(collection))` or `items .∈ Ref(collection)`.
